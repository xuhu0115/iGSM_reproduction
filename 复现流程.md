# 第二阶段-模型训练


创建了两个文件：

**[model.py](vscode-webview://0a2jmrpr62q1olsbb8o4tsmc1bevp1e1h18oe92gulh3ue3683s4/09_project/reading/Physics_of_Language_Models/iGSM/model.py)** — GPT-2 + RoPE 模型定义

- `precompute_freqs_cis` / `apply_rotary_emb`：RoPE 实现
- `GPTConfig`：模型超参数配置类
- `MODEL_CONFIGS`：10 个预定义变体（Size-1 和 Size-2）
- `GPT`：完整模型，含 causal attention、MLP、weight tying

**[train.py](vscode-webview://0a2jmrpr62q1olsbb8o4tsmc1bevp1e1h18oe92gulh3ue3683s4/09_project/reading/Physics_of_Language_Models/iGSM/train.py)** — 训练脚本

- 超参数完全遵循论文 Appendix F.1
- 支持读取预生成的 `.bin` 文件，或 on-the-fly 生成
- 支持单卡 / 多卡（`torchrun`）
- 自动保存 milestone checkpoint（step 10k/25k/50k/75k/100k）供后续 probing 使用

### 启动训练命令

```bash
conda activate llm
cd /path/to/iGSM

# 主实验：GPT2-12-12 on iGSM-med
python train.py --dataset med

# 使用多卡（4 GPU）
torchrun --nproc_per_node=4 train.py --dataset med

# depth 对比实验：所有 size-1 模型
for model in GPT2-4-21 GPT2-8-15 GPT2-12-12 GPT2-16-10 GPT2-20-9; do
    python train.py --dataset med --model $model
done
```

## 开始训练

**单卡（推荐先测试）：**
```bash
MASTER_PORT=29501 python train.py \
    --dataset med \
    --model GPT2-12-12 \
    --micro_batch 256 \
    --out_dir checkpoints/
```

此时 `accum_steps = 512/256 = 2`，速度是原来的 **~8倍**，约 **12 小时**。A800 80GB 跑 GPT2-12-12（85M参数）+ micro_batch=256 + ctx=768 的显存占用约 **18GB**，远未触顶。

**双卡（充分利用两块 A800）：**


```bash
torchrun --nproc_per_node=8 --master_port 29502 \
    train.py \
    --dataset med \
    --model GPT2-12-12 \
    --micro_batch 64 \
    --out_dir checkpoints \
    --bf16
```

**同时建议加 `--bf16` 参数**，A800 有原生 bf16 支持，比 fp16+GradScaler 更快且更稳定：
`--bf16` 和 `--compile` 默认已开启，无需额外指定。

|优化项|说明|预估加速|
|---|---|---|
|**修复 chunked lm_head**|从 97次 lm_head → 1次，消除最大瓶颈|**3-5x**|
|**bf16 替换 fp16**|去掉 GradScaler 开销，A800 原生支持|~10%|
|**torch.compile**|算子融合，消除 Python 开销|~20-30%|
|**TF32 matmul**|免费的矩阵乘法加速|~5-10%|
|**DataLoader prefetch**|消除 CPU-GPU 数据等待|~5%|

**综合预估：12小时 → ~7小时**（主要来自修复 lm_head 的循环调用）

**如果显存仍然不够**，继续缩小 micro_batch：

```bash
python train.py --dataset med --micro_batch 16 --out_dir checkpoints/
```


训练过程约 100k steps，每 100 步会打印一次 loss，每 5000 步保存一次 checkpoint。你会看到：

```
step      0/100000 | loss 10.8132 | lr 0.00e+00 | 0.0k tok/s
step    100/100000 | loss  8.3241 | lr 2.00e-04 | 45.2k tok/s
step    200/100000 | loss  6.1203 | lr 4.00e-04 | 46.1k tok/s
...
step  50000/100000 | loss  0.8321 | lr 1.20e-03 | 47.3k tok/s
...
```

loss 降到 0.5 以下说明训练正常。

## 关于训练时间

|硬件|预估时间（100k steps, batch=512, ctx=768）|
|---|---|
|单块 A100|~4 小时|
|单块 V100|~8 小时|
|单块 3090/4090|~6-10 小时|
|CPU only|不建议（极慢）|

**如果 GPU 显存不足**（如 <16GB），可以缩小 batch size：

```bash
python train.py --dataset med --model GPT2-12-12 \
    --batch 128 --out_dir checkpoints/
```

**多卡加速**（如 4 卡）：

```bash
torchrun --nproc_per_node=4 train.py --dataset med --out_dir checkpoints/
```

## 快速验证训练脚本可用

如果想先用少量 steps 验证整个流程通畅，可以：

```bash
python train.py --dataset med --steps 200 --batch 32 \
    --save_interval 100 --out_dir checkpoints_test/
```

几分钟内就能拿到一个小的 checkpoint，验证 evaluate.py 能正常读取。


# 第三阶段 评估

**[evaluate.py](vscode-webview://0a2jmrpr62q1olsbb8o4tsmc1bevp1e1h18oe92gulh3ue3683s4/09_project/reading/Physics_of_Language_Models/iGSM/evaluate.py)** — 批量推理 + 正确率评估

核心设计：

- `generate_test_problems(dataset, op, n)` — 用 `test_bin`（hash≥17 mod 23）生成 per-op 测试集，严格复现论文的 train/test 分离
- `batch_generate()` — 支持 beam=1 greedy 和 beam=4 multinomial sampling（与论文完全一致）
- `check_output()` — 调用 `true_correct(tokens, problem)` 做完整验证：答案正确性 + 计算步骤正确性 + 引用关系正确性
- Figure 4 所需统计：每个正确解中 `n_unnecessary_op`、`n_unnecessary_param`

**[plot_results.py](vscode-webview://0a2jmrpr62q1olsbb8o4tsmc1bevp1e1h18oe92gulh3ue3683s4/09_project/reading/Physics_of_Language_Models/iGSM/plot_results.py)** — 复现 Figure 3 & 4

## 运行步骤

```bash
conda activate llm
cd /path/to/iGSM

# Step 1: beam=1 评估（greedy）
python evaluate.py \
    --ckpt checkpoints/GPT2-12-12_med_latest.pt \
    --dataset med --p_format pq \
    --n_problems 4096 --beam 1 \
	--batch_size 128 \
    --save_json results/GPT2-12-12_med_pq_beam1.json

# Step 2: beam=4 评估（multinomial sampling）
CUDA_VISIBLE_DEVICES=1 python evaluate.py \
    --ckpt checkpoints/GPT2-12-12_med_latest.pt \
    --dataset med --p_format pq \
    --n_problems 4096 --beam 4 --dosample \
    --batch_size 128 \
    --save_json results/GPT2-12-12_med_pq_beam4.json

# Step 3: qp format 评估
python evaluate.py \
    --ckpt checkpoints/GPT2-12-12_med_latest.pt \
    --dataset med --p_format qp \
    --n_problems 4096 --beam 1 \
    --save_json results/GPT2-12-12_med_qp_beam1.json

# Step 4: 绘图
python plot_results.py --fig3 --fig4 \
    --json_pq results/GPT2-12-12_med_pq_beam1.json \
    --json_qp results/GPT2-12-12_med_qp_beam1.json \
    --dataset med --out figures/GPT2-12-12_med
```
